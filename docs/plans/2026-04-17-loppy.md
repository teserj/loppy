# Loppy Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Loppy Claude Code plugin — a scaffolding + operations layer for a Karpathy-style LLM wiki, with a pure-bash mechanics helper (`loppy`), a PreToolUse guard hook, and a single `/wiki <op>` slash command.

**Architecture:** Four-layer plugin: LLM (`/wiki` prompt), Schema (`wiki-schema.yaml`, `CLAUDE.md`), Guard (PreToolUse hook blocking destructive vault ops), Mechanics (`bin/loppy` bash subcommands), Data (vault at `$VAULT_DIR` with `sources/processed/` and `wiki/`).

**Tech Stack:** Bash (POSIX + bashisms), `jq`, `awk`, `find`, `git`, `bats-core` for tests. No Python, no Node, no LLM dependency in the helper.

**Spec:** `docs/specs/2026-04-17-loppy-design.md`

```bash
  export VAULT_DIR="$TEST_TMP/vault"
  export SOURCES_DIR="$VAULT_DIR/sources"
  export WIKI_DIR="$VAULT_DIR/wiki"

  export CONFIG_DIR="$XDG_CONFIG_HOME/loppy"

{
  "vault_dir": "$VAULT_DIR",
  "sources_dir": "$SOURCES_DIR",
  "wiki_dir": "$WIKI_DIR",
  "batch_size": 5
}
EOF
}

cleanup_env() {
  [[ -n "${TEST_TMP:-}" && -d "$TEST_TMP" ]] && rm -rf "$TEST_TMP"
}
```

```bash
#!/usr/bin/env bash
# loppy — mechanics helper for the Loppy Claude Code plugin.
# Pure bash. No LLM dependency.
set -euo pipefail

loppy_config_path() {
  echo "${XDG_CONFIG_HOME:-$HOME/.config}/loppy/config.json"
}

cmd_config() {
  local path; path="$(loppy_config_path)"
  if [[ ! -f "$path" ]]; then
    echo "loppy: config not found. Run setup.sh first." >&2
    return 1
  fi
  if [[ $# -eq 0 ]]; then
    jq . "$path" 2>/dev/null || { echo "loppy: malformed config JSON at $path" >&2; return 1; }
  else
    local key="$1"
    local value
    value="$(jq -er --arg k "$key" '.[$k]' "$path" 2>/dev/null)" || {
      echo "loppy: config key not found: $key" >&2
      return 1
    }
    echo "$value"
  fi
}

usage() {
  cat <<'EOF'
loppy — mechanics helper for the Loppy plugin.

Usage:
  loppy config [key]              Print config JSON or a single value.

EOF
}

main() {
  local sub="${1:-}"
  if [[ -z "$sub" || "$sub" == "--help" || "$sub" == "-h" ]]; then
    usage; return 0
  fi
  shift || true
  case "$sub" in
    config) cmd_config "$@" ;;
    *) echo "loppy: unknown subcommand: $sub" >&2; usage >&2; return 1 ;;
  esac
}

main "$@"
```

```bash
cmd_move() {
  local src="${1:-}"
  if [[ -z "$src" ]]; then
    echo "loppy move: source path required" >&2
    return 1
  fi
  if [[ ! -f "$src" ]]; then
    echo "loppy move: source not found: $src" >&2
    return 1
  fi
  local sources_dir; sources_dir="$(cmd_config sources_dir)"
  local vault_dir; vault_dir="$(cmd_config vault_dir)"
  local processed="$sources_dir/processed"
  local dest="$processed/$(basename "$src")"
  if [[ -e "$dest" ]]; then
    echo "loppy move: destination exists: $dest" >&2
    return 1
  fi
  mkdir -p "$processed"
  if git -C "$vault_dir" rev-parse --is-inside-work-tree >/dev/null 2>&1; then
    git -C "$vault_dir" mv "$src" "$dest"
  else
    mv "$src" "$dest"
  fi
}
```

```bash
# Map namespace → section heading used in index.md.
# Keeps insertion order stable.
_index_sections() {
  cat <<'EOF'
concepts|## Concepts
entities|## Entities
sources|## Sources
topics|## Topics
EOF
}

# Derive namespace from a wiki-relative path like "wiki/concepts/foo.md".
_namespace_of() {
  local p="$1"
  p="${p#wiki/}"
  echo "${p%%/*}"
}

cmd_index_merge() {
  local wiki_dir; wiki_dir="$(cmd_config wiki_dir)"
  local idx="$wiki_dir/index.md"
  [[ -f "$idx" ]] || { echo "loppy index-merge: $idx not found" >&2; return 1; }

  local today="${LOPPY_TODAY:-$(date +%Y-%m-%d)}"
  local tmpdir; tmpdir="$(mktemp -d)"
  local payload="$tmpdir/payload.json"
  cat > "$payload"  # read stdin

  # Validate JSON up front
  jq -e . "$payload" >/dev/null 2>&1 || {
    rm -rf "$tmpdir"
    echo "loppy index-merge: stdin is not valid JSON" >&2
    return 1
  }

  # Extract existing entries from the current index:
  #   path<TAB>summary
  # Matches lines of form: - [[wiki/<ns>/<slug>]] — <summary> (updated: YYYY-MM-DD)
  local existing="$tmpdir/existing.tsv"
  awk '
    /^- \[\[wiki\// {
      # capture the link target
      match($0, /\[\[([^]]+)\]\]/, m)
      link = m[1]
      path = link ".md"
      # strip leading "- [[...]] — "
      rest = $0
      sub(/^- \[\[[^]]+\]\][[:space:]]*—[[:space:]]*/, "", rest)
      # strip trailing " (updated: YYYY-MM-DD)"
      sub(/[[:space:]]*\(updated:[[:space:]]*[^)]+\)$/, "", rest)
      printf "%s\t%s\n", path, rest
    }
  ' "$idx" > "$existing"

  # Build the merged map: start from existing, overlay payload upserts.
  local merged="$tmpdir/merged.tsv"
  cp "$existing" "$merged"

  # Append/update from payload
  jq -r '.[] | [.path, .summary] | @tsv' "$payload" > "$tmpdir/payload.tsv"
  while IFS=$'\t' read -r path summary; do
    # Remove any existing row with this path, then append.
    grep -v $'^'"$path"$'\t' "$merged" > "$merged.new" || true
    mv "$merged.new" "$merged"
    printf "%s\t%s\n" "$path" "$summary" >> "$merged"
  done < "$tmpdir/payload.tsv"

  # Orphan detection: list every .md under wiki/ excluding index.md, log.md,
  # and wiki-schema.yaml. Compare against merged paths.
  local ondisk="$tmpdir/ondisk.txt"
  ( cd "$wiki_dir" && find . -type f -name '*.md' \
      ! -name 'index.md' ! -name 'log.md' \
      | sed 's|^\./|wiki/|' | sort ) > "$ondisk"
  local merged_paths="$tmpdir/merged_paths.txt"
  cut -f1 "$merged" | sort -u > "$merged_paths"
  while read -r p; do
    if ! grep -Fxq "$p" "$merged_paths"; then
      echo "loppy index-merge: orphan: $p" >&2
    fi
  done < "$ondisk"

  # Render new index.md
  local total; total=$(wc -l < "$merged" | tr -d ' ')
  local out="$tmpdir/index.md.new"
  {
    echo "---"
    echo "type: index"
    echo "title: Wiki Index"
    echo "updated: $today"
    echo "---"
    echo ""
    echo "# Wiki Index"
```

```bash
    return 1

  # the header and the rest.

  # Everything up to and including that line is the header.
  local sep_line
  sep_line=$(awk '
    found_h1 { if ($0 == "---") { print NR; exit } }
    /^# / { found_h1 = 1 }
  ' "$logf")
  if [[ -z "$sep_line" ]]; then
    # No separator — append a basic one, then the entry.
    cp "$logf" "$out"
    printf "\n---\n\n## [%s] %s | %s\n\n%s\n" "$today" "$op" "$title" "$body" >> "$out"
  else
    head -n "$sep_line" "$logf" > "$out"
    printf "\n## [%s] %s | %s\n\n%s\n" "$today" "$op" "$title" "$body" >> "$out"
    tail -n +$((sep_line + 1)) "$logf" >> "$out"
  fi
```

```bash
# Built-in schema defaults. Used when wiki-schema.yaml missing or for baseline rules.
_allowed_types()      { echo "entity concept source project thought todo worklog"; }
_allowed_domains()    { echo "tech finance business work career hobbies family parenting relationships self"; }
_allowed_confidence() { echo "high medium low stale"; }
_required_fields()    { echo "type title created updated confidence domain tags links"; }

# Extract YAML frontmatter as flat JSON. Supports:
#   key: value
#   key: [a, b, c]     (inline array)
# Lists starting with "- " are not supported here (not used in our schema).
_frontmatter_to_json() {
  local file="$1"
  awk '
    BEGIN { inside=0; first=0; printf "{" }
    /^---[[:space:]]*$/ {
      if (!first) { inside=1; first=1; next }
      else { inside=0; exit }
    }
    inside && /^[A-Za-z_-]+:/ {
      # split on first colon
      p = index($0, ":")
      key = substr($0, 1, p-1)
      val = substr($0, p+1)
      sub(/^[[:space:]]*/, "", val)
      sub(/[[:space:]]*$/, "", val)
      # strip surrounding quotes
      if (val ~ /^".*"$/) { val = substr(val, 2, length(val)-2) }
      if (emitted++) printf ","
      if (val ~ /^\[.*\]$/) {
        # inline array: parse elements
        inner = substr(val, 2, length(val)-2)
        gsub(/[[:space:]]*,[[:space:]]*/, "\",\"", inner)
        gsub(/"/, "\\\"", inner)       # escape any existing quotes
        if (inner == "") { printf "\"%s\":[]", key }
        else             { printf "\"%s\":[\"%s\"]", key, inner }
      } else {
        gsub(/\\/, "\\\\", val)
        gsub(/"/, "\\\"", val)
        printf "\"%s\":\"%s\"", key, val
      }
    }
    END { printf "}" }
  ' "$file"
}

# Compute day-difference between today and a YYYY-MM-DD string.
_days_since() {
  local d="$1"
  local today="${LOPPY_TODAY:-$(date +%Y-%m-%d)}"
  python3 - <<PY 2>/dev/null || _days_since_fallback "$d" "$today"
from datetime import date
a = date.fromisoformat("$d")
b = date.fromisoformat("$today")
print((b - a).days)
PY
}

_days_since_fallback() {
  # Pure bash fallback if python3 missing.
  local d="$1" today="$2"
  local d_epoch today_epoch
  d_epoch=$(date -d "$d" +%s 2>/dev/null) || return 0
  today_epoch=$(date -d "$today" +%s 2>/dev/null) || return 0
  echo $(( (today_epoch - d_epoch) / 86400 ))
}

cmd_lint_frontmatter() {
  local wiki_dir; wiki_dir="$(cmd_config wiki_dir)"
  local tmpdir; tmpdir="$(mktemp -d)"

  # Build set of all existing wiki page paths (relative, without .md) for link resolution.
  local existing="$tmpdir/existing.txt"
  ( cd "$wiki_dir" && find . -type f -name '*.md' \
      ! -name 'index.md' ! -name 'log.md' \
      | sed 's|^\./||; s|\.md$||' | sort ) > "$existing"

  # Extract entries from index.md for orphan detection.
  local index_paths="$tmpdir/index.txt"
  if [[ -f "$wiki_dir/index.md" ]]; then
    awk '
      /^- \[\[wiki\// {
        match($0, /\[\[([^]]+)\]\]/, m)
        sub(/^wiki\//, "", m[1])
        print m[1]
      }
    ' "$wiki_dir/index.md" | sort -u > "$index_paths"
  else
    : > "$index_paths"
  fi

  local findings_dir="$tmpdir/findings"; mkdir -p "$findings_dir"

  # Iterate every candidate page.
  while IFS= read -r rel; do
    local file="$wiki_dir/$rel.md"
    local fm; fm="$(_frontmatter_to_json "$file")"
    local findings='[]'

    # Helper: append a finding as JSON.
    add() {
      findings=$(echo "$findings" | jq --argjson f "$1" '. + [$f]')
    }

    # 1. required fields present?
    for field in $(_required_fields); do
      if ! echo "$fm" | jq -e --arg k "$field" 'has($k)' >/dev/null; then
        add "{\"level\":\"error\",\"rule\":\"missing-field\",\"field\":\"$field\"}"
      fi
    done

    # 2. enum checks (only if the field exists)
    local t; t=$(echo "$fm" | jq -r '.type // empty')
    if [[ -n "$t" ]] && ! echo " $(_allowed_types) " | grep -q " $t "; then
      add "{\"level\":\"error\",\"rule\":\"bad-enum\",\"field\":\"type\",\"value\":\"$t\"}"
    fi
    local dom; dom=$(echo "$fm" | jq -r '.domain // empty')
    if [[ -n "$dom" ]] && ! echo " $(_allowed_domains) " | grep -q " $dom "; then
      add "{\"level\":\"error\",\"rule\":\"bad-enum\",\"field\":\"domain\",\"value\":\"$dom\"}"
    fi
    local conf; conf=$(echo "$fm" | jq -r '.confidence // empty')
    if [[ -n "$conf" ]] && ! echo " $(_allowed_confidence) " | grep -q " $conf "; then
      add "{\"level\":\"error\",\"rule\":\"bad-enum\",\"field\":\"confidence\",\"value\":\"$conf\"}"
    fi

    # 3. staleness
    local upd; upd=$(echo "$fm" | jq -r '.updated // empty')
    if [[ -n "$upd" && "$conf" != "stale" ]]; then
      local age; age=$(_days_since "$upd")
      if [[ -n "$age" && "$age" -gt 90 ]]; then
        add "{\"level\":\"warn\",\"rule\":\"stale\",\"age_days\":$age}"
      fi
    fi

    # 4. orphan (not in index)
    if ! grep -Fxq "$rel" "$index_paths"; then
      add "{\"level\":\"warn\",\"rule\":\"orphan\"}"
    fi

    # 5. broken links
    local links; links=$(echo "$fm" | jq -r '.links // [] | .[]? // empty')
    while IFS= read -r link; do
      [[ -z "$link" ]] && continue
      local target="${link#wiki/}"
      if ! grep -Fxq "$target" "$existing"; then
        add "{\"level\":\"error\",\"rule\":\"broken-link\",\"target\":\"$link\"}"
      fi
    done <<< "$links"

    # Only include this page in output if it has findings.
    if [[ "$findings" != "[]" ]]; then
      echo "{\"path\":\"wiki/$rel.md\",\"findings\":$findings}" >> "$findings_dir/all.ndjson"
    fi
  done < "$existing"

  # Emit a JSON array.
  if [[ -f "$findings_dir/all.ndjson" ]]; then
    jq -s '.' "$findings_dir/all.ndjson"
  else
    echo "[]"
  fi
  rm -rf "$tmpdir"
}
```

```bash
  Bash)
    cmd="$(echo "$payload" | jq -r '.tool_input.command // empty')"
    # Allowlist: loppy subcommands and git mv/rm.
    if [[ "$cmd" =~ (^|[[:space:]])loppy[[:space:]] ]]; then exit 0; fi
    if [[ "$cmd" =~ (^|[[:space:]])git[[:space:]].+[[:space:]](mv|rm)([[:space:]]|$) ]]; then exit 0; fi

    # Block if command contains a destructive verb AND a vault path.
    if [[ "$cmd" =~ (^|[[:space:]])(rm|shred|rmdir|unlink|mv|dd)([[:space:]]|$) ]]; then
      if [[ "$cmd" == *"$vault_dir"* ]]; then
        echo "Blocked: destructive op on vault ('$vault_dir'). Use 'loppy move' or '/wiki'." >&2
        exit 2
      fi
    fi
    exit 0
    ;;
  Write)
    fp="$(echo "$payload" | jq -r '.tool_input.file_path // empty')"
    content="$(echo "$payload" | jq -r '.tool_input.content // empty')"
    if [[ "$fp" == "$vault_dir"* ]]; then
      # Check if content is empty or whitespace-only.
      if [[ -z "$content" ]] || [[ "$content" =~ ^[[:space:]]*$ ]]; then
        echo "Blocked: empty Write to vault path '$fp'. Refusing to truncate." >&2
        exit 2
      fi
    fi
    exit 0
    ;;
  *)
    exit 0
    ;;
esac
```

```bash
#!/usr/bin/env bash
# Loppy setup — one-time vault initializer.
set -euo pipefail

SELF_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
TEMPLATES="$SELF_DIR/templates"
CONFIG_DIR="${XDG_CONFIG_HOME:-$HOME/.config}/loppy"
CONFIG_FILE="$CONFIG_DIR/config.json"

prompt() {
  # prompt <message> <default>
  local msg="$1" default="${2:-}"
  local answer
  if [[ -n "$default" ]]; then
    printf "%s [%s]: " "$msg" "$default" >&2
  else
    printf "%s: " "$msg" >&2
  fi
  IFS= read -r answer || true
  if [[ -z "$answer" ]]; then answer="$default"; fi
  printf "%s" "$answer"
}

confirm() {
  # confirm <message>  (default N)
  local msg="$1" answer
  printf "%s [y/N]: " "$msg" >&2
  IFS= read -r answer || true
  [[ "$answer" =~ ^[Yy]$ ]]
}

echo "== Loppy setup ==" >&2

VAULT_DIR="$(prompt "Vault base directory" "$HOME/.vault")"
SOURCES_SUB="$(prompt "Raw sources subdir (relative to vault)" "sources")"
WIKI_SUB="$(prompt "Compiled wiki subdir (relative to vault)" "wiki")"
BATCH_SIZE="$(prompt "Batch size for ingest" "5")"

mkdir -p "$VAULT_DIR"
SOURCES_DIR="$VAULT_DIR/$SOURCES_SUB"
WIKI_DIR="$VAULT_DIR/$WIKI_SUB"
mkdir -p "$SOURCES_DIR/processed" "$WIKI_DIR"

# Git init prompt if not already a repo.
if ! git -C "$VAULT_DIR" rev-parse --is-inside-work-tree >/dev/null 2>&1; then
  if confirm "Initialize git repo in $VAULT_DIR?"; then
    git -C "$VAULT_DIR" init -q
  else
    echo "  (skipped git init — deletions won't be recoverable)" >&2
  fi
fi

# Write config JSON.
mkdir -p "$CONFIG_DIR"
cat > "$CONFIG_FILE" <<EOF
{
  "vault_dir": "$VAULT_DIR",
  "sources_dir": "$SOURCES_DIR",
  "wiki_dir": "$WIKI_DIR",
  "batch_size": $BATCH_SIZE
}
EOF
echo "  wrote $CONFIG_FILE" >&2

# Copy wiki-schema.yaml (confirm if exists).
SCHEMA_DEST="$WIKI_DIR/wiki-schema.yaml"
if [[ -f "$SCHEMA_DEST" ]]; then
  if confirm "$SCHEMA_DEST exists. Overwrite?"; then
    cp "$TEMPLATES/wiki-schema.yaml" "$SCHEMA_DEST"
    echo "  updated $SCHEMA_DEST" >&2
  else
    echo "  kept existing $SCHEMA_DEST" >&2
  fi
else
  cp "$TEMPLATES/wiki-schema.yaml" "$SCHEMA_DEST"
  echo "  created $SCHEMA_DEST" >&2
fi

# Copy index.md and log.md only if missing (never clobber).
for f in index.md log.md; do
  if [[ ! -f "$WIKI_DIR/$f" ]]; then
    cp "$TEMPLATES/$f" "$WIKI_DIR/$f"
    echo "  created $WIKI_DIR/$f" >&2
  else
    echo "  kept existing $WIKI_DIR/$f" >&2
  fi
done

echo "Done." >&2
```

````markdown
---
description: Ingest sources, query the wiki, or lint it.
argument-hint: ingest|query|lint [args]
---

You are operating the Loppy wiki — a Karpathy-style LLM-maintained knowledge base. Your job is to run the operation specified by $ARGUMENTS and report back cleanly.

## Always start with

1. Run `${CLAUDE_PLUGIN_ROOT}/bin/loppy config` and store the JSON (vault_dir, sources_dir, wiki_dir, batch_size).
2. Parse $ARGUMENTS. First token = operation (`ingest`, `query`, `lint`). Remaining tokens = operation-specific args.

If the operation is unknown, print a one-line usage reminder and stop.

## ingest

User form A: `/wiki ingest` (no further args) → batch mode.
User form B: `/wiki ingest <path>` → single-entry mode for that source file.

### Batch mode workflow

1. Read `wiki/index.md` and `wiki/wiki-schema.yaml` once.
2. Call `loppy next` to get up to `batch_size` unprocessed source paths.
   - If empty, tell the user "Nothing to ingest." and stop.
3. For each source path, in order (never in parallel — parallel writes corrupt files):
   a. Read the source file.
   b. Decide: new source page + which concept/entity/topic pages to create or update.
      Target 5–15 page touches per source. Write summary-level pages in `wiki/sources/`;
      write synthesis pages in `wiki/concepts/`, `wiki/entities/`, `wiki/topics/` as appropriate.
   c. Every new page must have full frontmatter per `wiki-schema.yaml` (type, title,
      created, updated, confidence, domain, tags, links). Copy today's date.
   d. Use Write for new files, Edit for existing ones. Do not run `rm`/`mv` directly —
      the guard hook will block it anyway.
4. Build a JSON array of `{"path": "wiki/<ns>/<slug>.md", "summary": "<one-line>"}` for every
   page you touched in step 3. Pass it to `loppy index-merge` via stdin:

   ```bash
   echo '[...]' | ${CLAUDE_PLUGIN_ROOT}/bin/loppy index-merge
   ```

   If stderr reports orphan warnings, that means a page exists on disk but you forgot to
   supply a summary. Write the summary, re-invoke `index-merge`. Bound this to one retry.

5. Append a log entry:

   ```bash
   ${CLAUDE_PLUGIN_ROOT}/bin/loppy log ingest "<short title>" <<EOF
   <body describing what was ingested, pages created/updated, notable new clusters>
   EOF
   ```

6. Move each successfully-ingested source into `processed/`:

   ```bash
   ${CLAUDE_PLUGIN_ROOT}/bin/loppy move "<absolute path>"
   ```

7. Report a one-paragraph summary to the user.

### Single-entry mode

Same as batch mode starting at step 2c, but skip `loppy next` — the user provided the path.

## query

Arg: a user question. Example: `/wiki query "what is the L1/L2 cache routing rule?"`

1. Read `wiki/index.md`. Scan by keyword + namespace for candidate pages.
2. Read the top few candidates in parallel.
3. Synthesize an answer. Cite using wiki-link syntax like [[wiki/concepts/llm-wiki-pattern]].
4. Offer to file the answer back as a new wiki page. If the user agrees, fall through to
   single-entry ingest mode using the Q&A as the source.

(Future: when a retrieval backend like qmd lands, the scan step swaps per config key
`retrieval_backend`. For v1, plain keyword+namespace is enough.)

## lint

1. Run `${CLAUDE_PLUGIN_ROOT}/bin/loppy lint-frontmatter` to get deterministic findings
   (missing fields, bad enums, staleness, orphans, broken links).
2. Group by severity. Report the count to the user up front.
3. For each **error**, propose a fix:
   - missing field → infer from content or tags, ask the user to confirm
   - bad enum → suggest the closest allowed value
   - broken link → suggest the closest existing target or propose removal
4. For each **stale** warning, ask the user per page whether to:
   - flip `confidence: stale` in frontmatter
   - refresh (fall through to single-entry ingest)
   - skip
5. Judgment layer (no helper can do this): spot-check a few pages for internal consistency,
   cross-ref density, duplicates, or contradictions. Report anything worth human attention.
6. For any frontmatter you edited, call `loppy index-merge` with updated summaries where
   relevant, and `loppy log lint "<title>"` summarizing the run.

## Hard rules

- Never run `rm`, `mv`, `shred`, `rmdir`, `unlink`, or `dd` against the vault. Use `loppy move`
  or `git rm`/`git mv` instead. The guard hook will block you otherwise.
- Never run two ingest/lint flows in parallel in the same vault. Single-threaded only.
- Never write an empty file into the vault — the guard hook blocks that too, but don't try.
- Summaries for `loppy index-merge` must be single-line, <200 chars, no Markdown formatting
  other than inline backticks.
````

```markdown
# Loppy — guidance for AI agents

This plugin owns a Karpathy-style LLM wiki. Read this before operating.

## Four layers

- **LLM (you):** via the `/wiki` slash command only.
- **Schema:** `wiki-schema.yaml` inside the user's vault — defines allowed type/domain/confidence values, required frontmatter fields, naming conventions.
- **Guard (PreToolUse hook):** blocks `rm`, `shred`, `rmdir`, `unlink`, `mv`, `dd` against any vault path. Blocks empty `Write` into the vault. Allowlist: `loppy <subcmd>`, `git mv`, `git rm`.
- **Mechanics (`bin/loppy`):** pure bash. Subcommands: `config`, `next`, `move`, `index-merge`, `log`, `lint-frontmatter`.

## Where to find things

- Config: `~/.config/loppy/config.json`. Call `loppy config` from a shell; never parse it yourself.
- Vault: paths are in the config — `vault_dir`, `sources_dir`, `wiki_dir`.
- Sources layout: raw files directly under `sources_dir/`; already-ingested sources live under `sources_dir/processed/`.
- Wiki layout: `wiki/index.md`, `wiki/log.md`, `wiki/wiki-schema.yaml`, plus per-namespace subdirs.

## Rules (not suggestions)

1. **Never run destructive shell ops on the vault.** The guard hook will block you. Use `loppy move` or `git mv`/`git rm`.
2. **Never parallelize wiki writes.** Parallel agent writes have been observed to corrupt files (umlauts → ASCII, per Gökçe). Single-threaded ingest and lint only.
3. **Always update the index.** Any page you create or meaningfully change must be included in the next `loppy index-merge` call with a one-line summary.
4. **Treat `loppy` as a Unix filter.** Pure bash, deterministic, no LLM. If a subcommand exits non-zero, read stderr and respond to it — don't ignore it.
5. **Sources move only after wiki is updated.** The `/wiki ingest` flow writes pages and updates index before calling `loppy move`. A crash mid-flow leaves sources in place — that is correct, not a bug.

## When to use `/wiki` vs direct file access

Use `/wiki ingest` / `query` / `lint` for anything that modifies the wiki or retrieves from it.
Use direct Read (and Grep if you must) for read-only exploration that isn't a user-facing query.
```
